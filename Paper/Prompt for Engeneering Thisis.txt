VS Code Project:

This is a Python-based web application for controlling and viewing TP-Link Tapo cameras.

Project Summary
Name: Tapo Camera Integration
Objective: To provide a web interface for interacting with TP-Link Tapo cameras on a local network.
Core Functionality:
Live video streaming to a web browser.
Pan/Tilt (PTZ) camera controls.
Displaying motion detection status.
Starting and stopping video recordings.
A web page to browse saved recordings.
Authentication: The application connects to the camera using credentials specified in a config.json file. The README.md notes two possible authentication methods: using local camera credentials or using admin with the TP-Link cloud password.
Technical Stack
Backend: Python 3.13, Flask
Real-time Communication: Flask-SocketIO with Eventlet for handling the video stream.
Camera Interaction:
pytapo: A Python library for controlling Tapo cameras (PTZ, settings).
opencv-python: Used for capturing and processing the RTSP video stream.
Frontend: HTML templates managed by Flask (Jinja2), with JavaScript for SocketIO communication and UI interaction.
Dependency Management: uv is used with pyproject.toml and a uv.lock file for reproducible dependency installation.
Containerization: A Dockerfile is provided for building and running the application in a Docker container.
Project Structure
run.py: The main entry point to start the Flask application.
app: The main Flask application module.
__init__.py: Initializes the Flask app, Flask-SocketIO, and loads configuration.
routes.py: Defines the main web routes, including the index page (/) and the video feed socket events.
recordings_routes.py: Defines routes for viewing and managing recordings.
camera.py: Contains the Camera class that interfaces with pytapo for controls and opencv for the video stream.
config.py: Handles loading settings from config.json.
templates/: Contains the Jinja2 HTML templates for the UI (index.html, recordings.html).
config.json: A user-provided configuration file (ignored by git) containing camera credentials and IP address.
pyproject.toml: Defines project metadata and dependencies.
uv.lock: A lockfile for reproducible builds with uv.
Dockerfile: Defines the steps to build a Docker image for the application.
README.md: Contains detailed instructions for setup, configuration, and troubleshooting.
video_stream.py: A standalone script that appears to be for reverse-engineering or directly accessing the camera's encrypted stream via HTTP, separate from the main Flask application's RTSP method. It uses pwntools and Crypto for decryption.
Paper: Contains LaTeX source files for what appears to be a thesis or academic paper about this project.
